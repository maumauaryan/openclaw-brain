# Auto-Memory System Rules

## Write Triggers

Every interaction with Aryan triggers memory write IF:
1. **Decision made** — pricing, positioning, offer structure
2. **Pattern observed** — what works / what fails
3. **Error occurred** — what went wrong + fix applied
4. **Success confirmed** — DM replied, hook performed, out reached
5. **New knowledge** — framework, insight, observation

## Memory Tiers

### Tier 1: Daily Log (memory/YYYY-MM-DD.md)
- Every interaction timestamped
- Context: what happened, what was decided
- Meta: who, when, what platform
- Format: Chronological raw entries

### Tier 2: Curated Memory ([[MEMORY]])
- High-signal insights only
- Patterns that apply broadly
- Decisions that guide future work
- Framework worth reusing

### Tier 3: Self-Improvement (SKILL.md)
- When pattern confirmed → update SKILL.md
- Tag with: `// Pattern confirmed [date]`
- Rewrites after Sunday synthesis

## Logging Format

```markdown
## [HH:MM UTC] [Platform] [Type]

**Context:** Who/what/where
**Input:** What Aryan said / what I observed
**Decision:** What was decided / what action taken
**Outcome:** Result (if known)
**Pattern:** If relevant, note the pattern for future use
```

## Autonomy Rules

- Never wait to write memory. Write immediately after insight.
- If memory file doesn't exist → create it.
- Always close session by updating memory/YYYY-MM-DD.md
- If quality < 7 → flag for Sunday synthesis

---

**Memory is continuous. Every turn is a learning opportunity.**
